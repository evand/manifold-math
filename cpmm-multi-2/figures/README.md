# Figures — the evidence, drawn

Companion figures to [`../rationale.md`](../rationale.md) — browse them all at
**<https://evand.github.io/manifold-math/>**. Humans are bad at
formulas; these translate the mechanism arguments into pictures. Every figure
is generated by a standalone script that computes through the
[reference implementation](../reference/manifold/) (the same oracle the proofs
use) plus direct ports of the vendor creation/liquidity rules, and **asserts
its headline numbers** — a figure cannot silently drift from the math.

## The figures

| Figure | Question it answers | Key result |
|---|---|---|
| [`fig_creation_cost`](fig_creation_cost.py) | What does it cost a creator to OPEN a market at the prices they believe? (n = 5 and 20) | v1: ante + forced self-trade (M2,274 at q=0.9 vs v2's flat M1,000) **plus** an outcome-dependent final-cost band — v1 makes the creator take a position; v2 doesn't. |
| [`fig_creation_cost_overlay`](fig_creation_cost.py) | Does that depend on the answer count? | Barely — the v1 penalty tracks the target price, not n (same script, second output). |
| [`fig_slippage`](fig_slippage.py) | The market shows q — how far does a trader's M100 move it? (equal creator budget) | At equal total outlay, v1 is thinner than v2 in **both** directions at every q (down-impact 22.8pt vs 15.5pt at q=0.9). |
| [`fig_liquidity_add_burn`](fig_liquidity_add_burn.py) | If I add M1,000 of liquidity, how much becomes liquidity? | One pool: v1 destroys \|2q−1\| in EV (M800 at a 90% answer). Sum-to-one whole-market add: lossless only at the uniform profile — M807/M1,000 EV loss with a 90% answer (n=5). v2: zero, everywhere. |
| [`fig_liquidity_add_split`](fig_liquidity_add_split.py) | Where does a whole-market add land, pool by pool? Is a ruled-out answer (q = 0.5%) a mana sink? | In EV at add-time prices: v1's landed mix is distorted *and* burned (90% favorite: its pool gets M96 of M1,000, M807 destroyed in EV — realized is outcome-dependent; a ruled-out answer still burns M140). v2's √variance merge (GP17) allocates ≈ in proportion to probability, everything landing in every resolution — a ruled-out answer takes M2 of M1,000, like adding to the (n−1)-answer market. |
| [`fig_other_split`](fig_other_split.py) | Splitting a new answer out of "Other" — who moves? | v1 drags bystanders 20% → 13.6% and the new answer *lands* at 35.5% (creator has no say); v2: bystanders byte-identical, new answer starts at the chosen price. |
| [`fig_other_split_sweep`](fig_other_split_sweep.py) | How does that damage depend on Other's price? | Absolute damage peaks mid-range (Σ-overshoot maxes at 1.5); relative damage worst at extremes; realistic worst case sits at q_O ≈ 1/n, where a traded market's pools are shallowest. |

All scenario numbers use **Manifold defaults** (`common/src/economy.ts` +
`tier.ts`): base liquidity tier, ante `max(numAnswers × M25, M100)`,
add-answer cost M25. Where magnitudes scale with pool depth, the scripts note
the M1,000-tier numbers.

## Regenerating

```bash
pip install -r ../requirements.txt   # numpy, sympy, pytest, matplotlib
for f in fig_*.py; do python "$f" || break; done
```

Each script writes `<name>.svg` (canonical, text preserved) and `<name>.png`
(embedded in the markdown docs), runs its asserts, and prints a console
summary of the headline numbers.

## Shared machinery

- [`figstyle.py`](figstyle.py) — one visual language for every figure:
  **v1 = gray, v2 = blue**, always; CVD-validated palette, recessive grid.
- [`scenarios.py`](scenarios.py) — vendor creation-rule ports
  (`cpmmMulti2SumToOnePools`, v1 uniform pools), general-profile v1 self-trade
  through the auto-arb simulator, the sum-to-one liquidity-add port with
  discard tracking, and a CLI for exploring any custom case:

  ```bash
  python scenarios.py --n 5 --featured 0.9 --ante 1000
  python scenarios.py --profile 0.55,0.25,0.1,0.06,0.04 --ante 2000
  ```
